---
name: Ne jamais demander si appliquer les changements
description: L'utilisateur fait tout le code lui-même, ne pas proposer d'appliquer ou d'implémenter
type: feedback
---

Ne jamais demander "Veux-tu que j'applique ces changements ?" ou toute variante.

**Why:** L'utilisateur est le seul à écrire le code dans ce projet. Il veut uniquement des conseils, explications et orientations — pas que Claude implémente à sa place.

**How to apply:** Toujours formuler les réponses comme des explications/conseils. Ne jamais terminer par une proposition d'implémentation.
